# 岗位图谱接入后端说明

## 一、前端数据结构（当前约定）

图谱页面使用的数据结构如下，后端接口只需返回**同构 JSON** 即可无缝接入。

```ts
// 单条转岗建议
interface TransferOption {
  title: string   // 岗位名称
  desc: string    // 简短描述
}

// 图谱上的一个岗位节点
interface CareerNode {
  id: string
  title: string
  hasTransfer?: boolean           // 是否展示「可转岗」入口
  transferOptions?: TransferOption[]  // 点击后展示的转岗选项列表
}

// 一条职业线（如产品线、技术线）
interface CareerTrack {
  id: string      // 唯一标识，如 product / tech
  name: string    // 展示名（可选，前端 tab 目前用 nodes[0].title）
  nodes: CareerNode[]  // 沿藤蔓从上到下顺序的节点
}
```

接口返回：**数组** `CareerTrack[]`，即多条职业线。

---

## 二、后端需要做的事

### 1. 新增接口（推荐）

- **路径**：`GET /job/career-map` 或 `GET /map/tracks`
- **鉴权**：可选（公开图谱可不鉴权）
- **响应体**：与上面 `CareerTrack[]` 一致，例如：

```json
[
  {
    "id": "product",
    "name": "产品线",
    "nodes": [
      { "id": "p1", "title": "产品实习生" },
      { "id": "p2", "title": "产品助理" },
      {
        "id": "p3",
        "title": "产品经理",
        "hasTransfer": true,
        "transferOptions": [
          { "title": "运营经理", "desc": "从幕后到台前" },
          { "title": "项目经理", "desc": "侧重落地执行" }
        ]
      }
    ]
  }
]
```

### 2. 数据从哪来（可选方案）

| 方案 | 说明 |
|------|------|
| **A. 新表** | 新建 `career_track`、`career_node`、`career_transfer_option`（或节点/转岗存 JSON），由运营或后台配置。 |
| **B. 基于岗位库** | 在现有 `job_position` 或岗位库上增加「线路、顺序、是否可转岗、转岗目标」等字段或关联表，由后端组装成上述结构。 |
| **C. 混合** | 线路与节点来自配置表，转岗目标引用岗位 ID，接口层拼成 `TransferOption[]`。 |

只要最终 HTTP 返回的 JSON 是 `CareerTrack[]` 即可，前端无需改结构。

## 三、数据从哪来：xls + DB + 可选 AI

- **原始数据**：`docs/dev-ops/doc/a13基于AI的大学生职业规划智能体-JD采样数据.xls`（岗位名称、薪资、公司等），已可通过 `import_jobs.py` 导入到 `job_position` 表。
- **图谱结构**：升岗/转岗数据存在 **career_track / career_node / career_transfer_option** 三张表（见 `data/migration/V004__career_map_tables.sql`），种子数据见 `V005__career_map_seed.sql`（与前端预设一致）。
- **AI 建议**：脚本 `docs/dev-ops/scripts/suggest_career_map.py` 可：
  1. 从上述 xls 抽取「岗位名称」去重；
  2. 可选调用 OpenAI，根据岗位名建议「职业线 + 升岗顺序 + 转岗选项」；
  3. 输出 JSON 供审阅，或加 `--write-db` 写入上述三张表。

使用前：`pip3 install xlrd openai pymysql`，设置 `OPENAI_API_KEY` 后执行：

```bash
cd docs/dev-ops/scripts
python3 suggest_career_map.py --ai              # 生成 career_map_suggested.json
python3 suggest_career_map.py --ai --write-db   # 审阅后写入 DB（慎用）
```

## 四、前端已做的对接准备

- **API**：`src/api/map.ts` 中已提供 `getCareerMapAPI()`，请求 `GET /job/career-map`。
- **页面**：`pages/map/index.vue` 进入时先调该接口；若返回成功且 `data.length > 0` 则用后端数据，否则使用本地 `career-map-data.ts` 的预设数据。
- **类型**：`career-map-data.ts` 中的 `CareerTrack`、`CareerNode`、`TransferOption` 与上述约定一致，后端按该结构返回即可。

后端接口上线后，只需保证路径与文档一致（或在前端 `map.ts` 里改一次 URL），无需再改页面逻辑。

---

## 五、后端已实现

- **GET /job/career-map**：从 `career_track` / `career_node` / `career_transfer_option` 组装并返回 `CareerTrack[]`。
- 执行迁移 `V004__career_map_tables.sql` 与种子 `V005__career_map_seed.sql` 后即可使用；之后可用 `suggest_career_map.py --ai` 生成建议并审阅后写入 DB。
