# 岗位图谱与 AI 接入：当前可做事项

> 在尚未接入本地大模型之前，有哪些事情可以立刻做；接入/微调大模型后，再回来测试哪些内容。

---

## 一、整体理解

- **当前状态**：岗位图谱的后端（DB + 接口 + 前端对接）已经打通。
- **后续方向**：若要「接入 AI 大模型并写入数据库」，主要工作在于：本地部署大模型、可选微调、再通过脚本（如 `suggest_career_map.py`）调用模型生成/更新图谱数据，然后回归测试。
- **不必等**：在捣鼓本地部署和微调之前，有很多事情可以现在就做，且对比赛和后续评测很有用。

---

## 二、现在就可以做的事（不依赖本地大模型）

### 2.1 跑通「纯 DB 版」岗位图谱

1. 执行迁移脚本：
   - `data/migration/V004__career_map_tables.sql`
   - `data/migration/V005__career_map_seed.sql`
2. 启动后端，确认：
   - `GET /job/career-map` 能返回 seed 数据；
   - 前端岗位图谱页已使用该接口数据（即使用 seed 的「真实后端图谱」）。
3. 效果：即使用不上 AI，图谱也能稳定跑起来，答辩时可展示完整链路。

### 2.2 做一版 job_family / job_level 规则

1. 按《岗位图谱与个性化推荐方案》中的思路，在后端或脚本里：
   - 写一个方法：根据 `job_position.title`（必要时加 `industry`）判断 `job_family`；
   - 写一个方法：从 title 解析 `job_level`（实习/初级/中级/经理/总监等）；
   - 对现有 JD 跑一遍，把结果导出到 CSV 或打印，先看效果。
2. 作用：为后续「从 JD 自动生成/更新图谱」打基础，也可在答辩中单独展示「岗位归类和级别解析规则」。

### 2.3 搭好评测集与比赛指标的骨架

1. 用脚本在 JD 上跑规则，先自动打 family / level / skills 的**伪标签**；
2. 从中抽样 100～300 条，人工修正为金标，存成 `evaluation_set.json`（格式见《岗位图谱与个性化推荐方案》第十章）；
3. 写一个简单的 `evaluate.py`：
   - 输入：`evaluation_set.json` + 一个「预测函数」（现阶段可先用规则版）；
   - 输出：job_family 准确率、job_level 宏平均 F1、技能标签 F1 等。
4. 以后只需把「预测函数」从规则换成模型调用，就能得到**微调前后对比**的量化结果。

> 这一步完全不依赖大模型，但对比赛展示非常关键：评测框架和指标先搭好，后续换模型即可出「提升了多少」的图表。

## 三、开始捣鼓本地大模型时要做的事（顺序参考）

等你有精力部署/微调本地大模型时，大致顺序可以是：

1. **本机拉起模型**  
   例如：`ollama run qwen2.5:7b-instruct`，或 vLLM 部署 Qwen。

2. **写一个极简的 Python 封装**（如 `local_llm.py`）  
   - 提供：`extract_tags_from_jd(text) -> {family, level, skills...}`；  
   - 可选：`explain_recommendation(user, jobs) -> explanation_json`。

3. **把评测脚本里的「预测函数」换成调用上述封装**  
   重新跑评测，得到「模型版」的分数，与规则版对比。

4. **再考虑 LoRA 微调**  
   用你人工修好的几百条样本做 SFT，让模型在该任务上更稳定；微调后再跑同一套评测，得到「微调前后」的对比。

5. **回归测试**  
   - 用 `suggest_career_map.py --ai` 生成 `career_map_suggested.json`，审阅后可选 `--write-db` 写入 `career_*` 三张表；  
   - 确认前端岗位图谱页展示的是 DB 数据，且与预期一致。

---

## 四、若只选两件「立刻动手」的事

1. **跑通 seed 图谱 + 接口联调**  
   执行 V004、V005，启动后端，确认图谱页拿到的是真实接口数据，而不是纯前端 mock。

2. **搭好评测集和 evaluate 脚本的骨架**  
   哪怕现阶段只评测「规则版」，你也已经具备：  
   - 明确的评测任务与数据格式；  
   - 可重复的指标（accuracy、F1）；  
   之后只需把预测逻辑换成模型调用，就能直接产出「模型/微调提升了多少」的图表和结论。
